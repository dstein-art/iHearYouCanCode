Getting Started

iHearYouCanCode — by David Stein
Set up p5.js and p5.sound — then make your first sound with the mouse
↗ p5.sound Docs

1. Add the script tags

Every exercise on this site runs in the browser. Add these lines to your HTML <head>. Load p5.js first, then p5.sound. 

<!-- Google icon font (optional — used by ProControls) -->
<link href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined&display=block" rel="stylesheet">

<!-- p5.js -->
<script src="https://cdn.jsdelivr.net/npm/p5@1.11.13/lib/p5.min.js"></script>

<!-- p5.sound (requires p5.js above) -->
<script src="https://cdn.jsdelivr.net/npm/p5@1.11.13/lib/addons/p5.sound.min.js"></script>

<!-- ProControls — needed from section 4 onward -->
<script src="https://cdn.jsdelivr.net/gh/dstein-art/proControls4p5js@v0.8.1/ProControls.js"></script>

<!-- Your sketch -->
<script src="sketch.js"></script>
Sections 2–3 need only p5.js and p5.sound. Section 4 adds ProControls for hardware-style UI — see the ProControls docs.

Audio context

Browsers block audio until the user clicks. userStartAudio() in mousePressed() unlocks the Web Audio context — use osc.started to check whether the oscillator is running.

Build on the sketch above: map mouseX to pitch and mouseY to volume in mouseMoved(). Click toggles the oscillator on and off.

4. Modify your sketch to use ProControls

Swap mouse mapping for an XYPad — horizontal for frequency, vertical for volume — plus a Selector to step through waveform types. Set ControlStyle before creating controls, call proControlBackground() in draw(), and wire onChange to update the oscillator.

5. Modify your sketch to control ADSR and oscillators

Connect a p5.Envelope to the oscillator with osc.amp(env). Use a second MultiSlider for attack, decay, sustain, and release — wire it to env.setADSR() and an ADSRDisplay panel so the envelope shape updates live. Click the canvas to trigger env.play(osc).
